PARTIE 1

Collecte automatisée

Scraping historisé respectant robots.txt avec reprise automatique. 66 864 avis + enrichissement mensuel via API Wikipédia (SIREN, CA, logos).

PARTIE 2

Architecture multi-bases

MongoDB pour les avis JSON (schémaless, volumineux) et PostgreSQL pour les métadonnées structurées + backend MLflow. Déployé via Docker Compose.

PARTIE 3

NLP & Machine Learning

BERT multilingue pour annoter la vérité terrain. Benchmarking LinearSVC vs LogisticRegression vs RandomForest. Suivi MLflow sur 6 modèles.

PARTIE 4

API & Mise en production

FastAPI REST sécurisée JWT (3 modes : Off/Partial/Full). 17 tests Pytest — 100% succès, ~78% de couverture. Interface Streamlit connectée.

PARTIE 5

Plateforme MLOps

Airflow orchestre scraping → insertion → ML. CI/CD GitLab (lint + tests). Grafana/Prometheus supervise 17 conteneurs Docker en temps réel.

PARTIE 6

Perspectives & améliorations

Intégration de modèles Transformer (CamemBERT, RoBERTa) pour affiner l'analyse. Extension à de nouvelles entreprises et sources (Google Reviews, Amazon). Alerting automatique en cas de data drift détecté.

LinearSVC

Meilleur global

Régularité sur les deux tâches. Exploite mieux la représentation TF-IDF bigrammes. Choix final retenu.

Classe neutre

Point difficile

Seulement 36.4% de rappel sur les avis neutres. Polarisation extrême des données (74.8% aux extrêmes) limite la nuance.

SMOTE abandonné

Décision technique

Le suréchantillonnage SMOTE a dégradé le F1-score. class_weight='balanced' retenu — plus efficace sans modifier le dataset.

05 · MLOps

Orchestration & supervision

AIRFLOW

Orchestration du pipeline

4 DAGs manuels (scraping → insertion → ML → global). BashOperator pour tâches légères, SSHOperator pour les modèles lourds sur Hyper-V.

GITLAB CI/CD

Automatisation qualité

Shell Runner local. 2 stages : lint (flake8) + tests API dans Docker. Script git_main.sh pour synchronisation dev → main et déclenchement CI.

GRAFANA / PROMETHEUS

Monitoring temps réel

4 dashboards : état global, ressources système, conteneurs Docker, métriques API. 17 conteneurs supervisés. Alertes CPU/RAM/latence.

MLFLOW

Traçabilité des modèles

Backend PostgreSQL. 6 runs trackés (3 sentiment + 3 note). Versioning des artefacts. Interface web sur :5000. Data drift documenté.
